## v0.2.1 — Bug fixes and advanced retrieval options

Bug fixes across the web interface, improved form validation, and new dataset upload support. Advanced retrieval options flags added but disabled by default. More fine-grained retrieval coming zoon.

### Bugfixes

#### Web Interface & Forms
- Sign-up form now keeps email/username on validation errors (no more retyping)
- Delete account (Danger Zone) bugs fixed—deletion now works properly
- Danger Zone UI redesigned for clarity and simplicity of use

#### Backend & Detection
- Detection results now ordered by anchor-target pairs (was inconsistent)
- Prompt loading supports alternative file paths for more flexible deployments

### New Capabilities

#### Dataset Management
- Dataset upload now available in the `Dataset` view of the webpage, upload in profile maintained for backwards compatibility

#### Advanced Retrieval (Opt-in)
New config flags (disabled by default):
- `use_cross_encoder_rerank` — rerank results with BAAI/bge-reranker-v2-m3
- `use_bidirectional_retrieval` — combine forward + reverse scoring (`bidirectional_alpha: 0.6`)

Enable in `config.yaml` to test; not recommended for production yet.

### Notes

- No breaking changes; backward compatible
- Drop-in replacement for v0.2.0-alpha
